# 中文注释规范

## 数据库

- 所有业务表必须使用 `COMMENT='中文表说明'`。
- 所有业务字段必须使用 `COMMENT '中文字段说明'`。
- 状态、类型、权限等枚举字段必须在注释中写明允许值和含义。
- 多租户、权限、版本、审计、软删除字段必须说明业务用途，不能只写字段名翻译。

## Java 代码

- Controller、Service、Mapper、DTO、Entity 必须有中文 JavaDoc，说明业务职责。
- DTO 和 Entity 的业务字段必须有中文字段注释。
- Service 的关键写操作必须说明权限规则、事务边界或服务端强制字段。
- Getter、Setter、简单构造器不强制逐个注释，避免无效噪音。
- 涉及 RAG 权限过滤、缓存 key、索引版本、引用追溯的代码必须解释安全原因。

## 后续交付要求

后续新增模块默认同步补齐中文注释，尤其是文档、切片、索引任务、检索、问答、评测相关代码和 SQL。
